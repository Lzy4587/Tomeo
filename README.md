## 🎨 UI/UX设计详解

### 1. 社交Feed流（Social Feed）

#### 设计理念
- **信息流浏览** - 类似Instagram/TikTok的垂直滚动feed
- **卡片式设计** - 每个视频作为独立卡片展示
- **清晰的层次结构** - 用户信息、视频内容、互动按钮分层展示

#### 理论依据
- **F型浏览模式** - 用户视线自然从左上到右下移动
- **卡片隐喻（Card Metaphor）** - 每个视频是一个可交互的独立单元
- **认知负荷理论** - 分页加载减少一次性信息过载

#### 实现细节
```cpp
// 视频卡片布局
用户头像 + 用户名 | 时间 | 更多按钮
-----------------------------------------
        视频缩略图 + 播放按钮
        (16:9比例，圆角设计)
-----------------------------------------
❤️ 点赞  💬 评论  🔗 分享  |  👁 观看数
-----------------------------------------
标题/描述文字（可折叠）
```

### 2. 互动按钮（Interaction Panel）

#### 设计原则
- **即时反馈** - 点击后立即视觉变化
- **清晰的状态** - 已点赞/未点赞用不同图标表示
- **触摸友好** - 按钮大小符合Fitts定律（最小44x44px）

#### 颜色语义
- ❤️ **红色心形** - 已点赞（情感化设计）
- 🤍 **空心心形** - 未点赞
- 💬 **对话气泡** - 评论入口
- 🔗 **链接图标** - 分享功能

### 3. 评论系统（Comment System）

#### 功能特性
1. **完整评论列表** - 滚动查看所有评论
2. **快捷评论** - 8种预设表情评论：
   - ❤️ 太棒了！
   - 🔥 超赞！
   - 😂 哈哈哈
   - 😍 好喜欢
   - 👏 厉害！
   - 💯 完美
   - ⭐ 点赞
   - 🎉 庆祝

3. **评论互动** - 可以对评论点赞
4. **时间显示** - 相对时间（如"5分钟前"）

#### 理论依据
- **减少摩擦力** - 快捷评论降低用户输入成本
- **社交证明** - 显示点赞数增强参与意愿
- **时间线可视化** - 相对时间更符合人类认知

### 4. 每日提醒（Daily Reminder）

#### BeReal风格特色
- **限时录制** - 倒计时显示营造紧迫感
- **随机时间** - 增加spontaneity（自发性）
- **简单操作** - 三个明确的选项按钮

#### 设计策略
```
[主标题] ⏰ Time to BeReal
[提示文字] 现在是录制视频的时刻！
[倒计时] 剩余时间: 2:00

[主按钮] 📸 立即录制（蓝色、突出）
[次按钮] 🕐 10分钟后提醒（边框样式）
[取消]    取消（文字按钮）

[设置区域]（可折叠）
```

#### 心理学原理
- **Scarcity Effect（稀缺效应）** - 限时促使行动
- **Commitment Consistency** - 设置习惯提高参与率
- **Social Pressure** - 知道朋友都在录制增强动力

## 🏗️ 技术架构

### 新增类图

```
社交功能模块
├── SocialTypes (数据结构)
│   ├── UserInfo - 用户信息
│   ├── Comment - 评论
│   ├── VideoPost - 视频帖子
│   └── DailyReminderSettings - 提醒设置
│
├── SocialManager (单例模式)
│   ├── 用户管理
│   ├── 帖子管理
│   ├── 互动功能
│   └── 数据持久化
│
├── UI组件
│   ├── VideoPostCard - 视频帖子卡片
│   ├── SocialFeedWidget - Feed流容器
│   ├── CommentDialog - 评论对话框
│   └── DailyReminderDialog - 每日提醒
│
└── 集成到主窗口
    └── MainWindow (tomeo.cpp)
```

### 数据流

```
用户操作 → UI组件 → SocialManager → 信号发射 → UI更新

示例：点赞流程
1. 用户点击❤️按钮
2. VideoPostCard::onLikeClicked()
3. SocialManager::likePost(postId)
4. 发射 postLiked(postId, true) 信号
5. 所有监听组件更新UI
```

## 🚀 编译和运行

### 前置要求
- Qt 5.15.2+
- C++11
- 操作系统：Windows/macOS/Linux

### 编译步骤

#### 方法1：Qt Creator
1. 打开 `tomeo.pro`
2. 配置项目
3. Build → Run

```bash

qmake tomeo.pro
make
./tomeo "/path/to/videos"

# Windows  
qmake tomeo.pro
nmake
tomeo.exe "C:\path\to\videos"
```
